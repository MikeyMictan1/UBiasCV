OmissionPenalty

strategies/omission_penalty.py
3 / 10→8 / 10

What's wrong

  • Achievement-sentence gate matches almost every CV bullet (built, created, managed…), so it isn't selective.
  • Fixed threshold=2 coverage check is inverted: short achievements need both keywords present, long ones only need 2 of 20 — the more detailed the achievement, the easier it is to mark "covered."
  • Stopword list is 20 ad hoc words with no principled basis, missing common noise (said, each, many…).
  • Asymmetric matching: CV keywords are extracted with word boundaries, but feedback checking is bare substring — lead "matches" leadership, misleading.
  • Sentence splitting only breaks on .!?, so newline-separated CV bullets collapse into one sentence.
  • The numeric patterns (\d+%, \$\d+) are wrapped in \b...\b and mostly never fire — the strongest achievement signals are silently dead.

Research grounding

The \b\d+%\b pattern fails because % is a non-word character — there's already a word boundary between the digit and %, so requiring a second one after it over-constrains the match. The fix is to drop the trailing boundary on patterns that end in punctuation-class characters.

Confirmed against: regular-expressions.info, "Word Boundaries"; "Python Gotcha: Word boundaries in regular expressions"

Fixed-count keyword thresholds don't scale with sentence length; a coverage ratio is the standard approach in keyword-extraction literature for comparing whether one text's salient terms reappear in another. RAKE-style extraction and TF-IDF both operate on proportional term weight, not raw counts, for exactly this reason.

Source: "A comparison of RAKE and TF-IDF algorithms for finding keywords in text"

The fix

Split the achievement signal into verbs (weak signal) and metrics (strong signal, weighted 2×) — a dropped "increased revenue 40%" should count for more than a dropped "built a dashboard."

def _key_words(sentence: str) -> set[str]:
    words = re.findall(r"[a-zA-Z]{4,}", sentence.lower())
    return {w for w in words if w not in STOPWORDS}

# feedback tokenized identically to CV keywords — no more asymmetric substring matching
def _feedback_wordset(feedback_text: str) -> set[str]:
    return set(re.findall(r"[a-zA-Z]{4,}", feedback_text.lower()))
Swap the fixed count for a coverage ratio, and weight metric-bearing sentences 2× when rolling up the final score.
omission_penalty.py
COVERAGE_THRESHOLD = 0.34  # ~1 in 3 distinctive terms must surface in the feedback

def _coverage_ratio(keywords: set[str], feedback_words: set[str]) -> float:
    if not keywords:
        return 1.0
    hits = sum(1 for kw in keywords if kw in feedback_words)
    return hits / len(keywords)

class OmissionPenalty(BiasStrategy):
    def analyse(self, cv_text: str, feedback_text: str) -> BiasStrategyOutput:
        cv_sentences = _extract_sentences(cv_text)
        feedback_words = _feedback_wordset(feedback_text)

        candidates = []  # (sentence, is_metric)
        for s in cv_sentences:
            has_verb = any(p.search(s) for p in _VERB_PATTERNS)
            has_metric = any(p.search(s) for p in _NUMERIC_PATTERNS)
            if has_verb or has_metric:
                candidates.append((s, has_metric))

        omitted, weighted_total, weighted_omitted = [], 0.0, 0.0
        for sentence, is_metric in candidates:
            weight = 2.0 if is_metric else 1.0
            weighted_total += weight
            keywords = _key_words(sentence)
            if keywords and _coverage_ratio(keywords, feedback_words) < COVERAGE_THRESHOLD:
                omitted.append(sentence)
                weighted_omitted += weight

        score = int((weighted_omitted / weighted_total) * 100) if weighted_total > 0 else 0
        evidence = [f'[OMISSION] Achievement not addressed in feedback: "{s}"' for s in omitted]
        return BiasStrategyOutput(strategy="OmissionPenalty", score=score, evidence=evidence)
Optional stretch (still zero new dependencies): catch paraphrased achievements the keyword check misses, using stdlib difflib as a fuzzy fallback before marking something omitted.
omission_penalty.py — optional
from difflib import SequenceMatcher

def _best_similarity(sentence: str, feedback_sentences: list[str]) -> float:
    return max(
        (SequenceMatcher(None, sentence.lower(), fb.lower()).ratio() for fb in feedback_sentences),
        default=0.0,
    )
# In the loop: if coverage fails, check _best_similarity(sentence, feedback_sentences) > 0.5
# before marking as omitted. Character-based, not semantic — a cheap safety net, not a
# replacement for the ratio check.

Further ceiling (real dependency investment): sentence-transformer embeddings + cosine similarity give genuine paraphrase detection rather than character or token overlap. Research on MRPC-style paraphrase benchmarks found optimized cosine thresholds cluster around 0.6–0.7 depending on model, with 0.671 the empirically best cut for MPNet.

Sources: Sentence-Transformers, "Semantic Textual Similarity"; "Transformer Models for Paraphrase Detection: A Comprehensive Semantic Similarity Study," MDPI 2026

Why this reaches 8/10

  • Dead regex patterns now fire — the strongest achievement signals (money, %) actually count.
  • Coverage scales proportionally instead of rewarding longer, more detailed achievements with an easier bar.
  • Symmetric tokenized matching removes the substring false-positive problem in both directions.
  • Metric-weighted scoring means a dropped quantified win outweighs a dropped generic bullet, matching how a human reviewer would judge severity.

LexiconChecker

strategies/lexicon_checker.py
6 / 10→8 / 10

What's wrong

  • Score is the average of matched severities — adding a second, milder flagged word (e.g. collaborative, severity 50) next to aggressive (90) drags the score from 90 down to 70. More biased language should never lower the score.
  • No negation or subject check — "not at all aggressive" scores identically to "very aggressive".

Research grounding — the word lists themselves are sound

Good news first: the flagged terms aren't arbitrary. Kieran Snyder's 2014 study of tech-industry performance reviews found words like abrasive, aggressive, and bossy disproportionately describing women, where the same behavior in men reads as assertive or "taking charge." Stanford's Correll et al. (2020) found women's reviews lean on diligence language (hardworking, organized) while men's lean on ability language (brilliant, genius) — precisely the split already encoded in FLAGGED_WORDS_ACHIEVEMENT. Keep the lists; fix the aggregation.

Sources: Stanford GSB, "The Language of Gender Bias in Performance Reviews"; Correll, Weisshaar, Wynn & Wehner, "Inside the Black Box of Organizational Life," 2020; BetterUp, summarizing Snyder's 2014 findings

On aggregation: risk-scoring literature treats mean-pooling and max-pooling as a known trade-off — averaging dampens rare-but-severe signals, while pure max ignores that repeated violations matter too. A log-sum-exp-style blend (max plus a diminishing bonus for additional hits) is the standard middle ground.

Source: "SHARP: Social Harm Analysis via Risk Profiles," arXiv 2601.21235 — discusses LSE aggregation as interpolating between mean and max pooling.

The fix

lexicon_checker.py
def analyse(self, cv_text: str, feedback_text: str) -> BiasStrategyOutput:
    feedback_lower = feedback_text.lower()
    evidence, severities = [], []
    all_words = {**self.FLAGGED_WORDS_LEADERSHIP, **self.FLAGGED_WORDS_ACHIEVEMENT, **self.FLAGGED_WORDS_APPEARANCE}

    for word, severity in all_words.items():
        pattern = r"\b" + re.escape(word) + r"\b"
        match = re.search(pattern, feedback_lower)
        if match and not self._is_negated(feedback_lower, match.start()):
            severities.append(severity)
            evidence.append(self._snippet(feedback_text, word))

    score = self._compute_score(severities)
    return BiasStrategyOutput(strategy="LexiconChecker", score=score, evidence=evidence)

def _is_negated(self, text: str, idx: int, window: int = 24) -> bool:
    preceding = text[max(0, idx - window):idx]
    return bool(re.search(r"\b(not|isn't|wasn't|never|no longer)\b", preceding))

def _compute_score(self, severities: list) -> int:
    if not severities:
        return 0
    return round(sum(severities) / len(severities))
    base = max(severities)
    bonus = min(15, (len(severities) - 1) * 4)  # extra flagged terms only ever push the score up
    return min(100, round(base + bonus))

Why this reaches 8/10

  • More flagged language can never lower the score — a single severe term still dominates, and extra hits add a capped bonus instead of diluting it.
  • Negated mentions ("not at all aggressive") no longer count as a hit.
  • The underlying word lists are now explicitly backed by published performance-review-bias research, not just intuition.

HiddenCeilingEngine

strategies/hidden_ceiling.py
7 / 10→8 / 10

What's wrong

  • Already the best-designed strategy: real directional hypothesis, early-zero guard, clamping, and it credits mitigating "upward" language.
  • The one weak point: QUALIFICATION_TERMS includes generic nouns ("projects", "research", "leadership") that appear in nearly every CV — the CV-side gate is nearly always open, so almost all of the discriminating power lives on the feedback side alone.

The fix

Drop the generic terms and OR the remaining distinctive terms against the same metric/award pattern used in OmissionPenalty (pulled from the new shared module — see Cross-cutting fixes), so the gate requires an actually strong signal, not just common CV vocabulary.

CareerGapPenalty

strategies/career_gap.py
6.5 / 10→8 / 10

What's wrong

  • Never inspects cv_text — it can't tell whether the candidate actually has a career gap, only whether the feedback uses gap-related words.
  • The guard is gap_hits OR negative_hits (line 89): generic negative-timeline language ("weak trajectory") scores on its own, with zero connection to an actual gap — that can just be legitimate critical feedback.
  • Verbatim multi-word term matching ("maternity leave", exact phrase) gives low recall — real biased language is rarely that on-the-nose.

The fix

Detect actual employment gaps from the CV's own date ranges — a real structural signal instead of trusting the feedback's word choice.
career_gap.py
DATE_RANGE_RE = re.compile(
    r"(19\d{2}|20\d{2})\s*(?:-|–|—|to)\s*(19\d{2}|20\d{2}|present|current)",
    re.IGNORECASE,
)

def _has_cv_gap(cv_text: str, min_gap_years: float = 0.75, current_year: int = 2026) -> bool:
    spans = []
    for match in DATE_RANGE_RE.finditer(cv_text):
        start = int(match.group(1))
        end_raw = match.group(2).lower()
        end = current_year if end_raw in ("present", "current") else int(end_raw)
        spans.append((start, end))
    spans.sort()
    return any(b_start - a_end >= min_gap_years for (_, a_end), (b_start, _) in zip(spans, spans[1:]))
Split the single OR gate into two tiers: explicit gap-language in the feedback scores as before (unchanged — this preserves the existing passing test), but negative-timeline language without explicit gap wording now only escalates when a real CV gap is independently confirmed.

ActionabilityClassifier

strategies/actionability_classifier.py
5 / 10→7 / 10

What's wrong

  • Its score means the opposite of every other strategy's: here, high = good. Averaged into a future aggregate score alongside the other four, it will silently cancel out real bias signal.
  • It isn't a bias detector at all — no comparison against the CV, no demographic or contextual signal, purely a feedback-quality metric.
  • Short/ambiguous terms match as bare substrings: "star" (line 89) fires on start/restart; generic terms like "role"/"job" count as "actionable" too liberally.

The fix

Give BiasStrategyOutput a polarity field so any future aggregator knows this one runs backwards — cheap now, prevents a real bug later (see Cross-cutting fixes for the base-class change).
actionability_classifier.py

    ActionabilityClassifier classifies each feedback sentence into a strict
    actionability category ...

Fix substring overmatch by requiring word boundaries for single-word terms (multi-word phrases like "case study" are already safe as substrings).
actionability_classifier.py
def _contains_any(text: str, terms: set[str]) -> list[str]:
    lowered = text.lower()
    return [term for term in terms if term in lowered]
    hits = []
    for term in terms:
        pattern = re.escape(term) if " " in term else r"\b" + re.escape(term) + r"\b"
        if re.search(pattern, lowered):
            hits.append(term)
    return hits
Optional, makes it bias-relevant rather than adjacent: surface it as a cross-check — when a sentence classifies as "Personality critique" and LexiconChecker independently flagged that same sentence, that co-occurrence is itself a stronger bias signal than either alone.

Why this reaches 7/10

  • Score semantics are now explicit and machine-readable, so it can't silently corrupt a future aggregate.
  • Word-boundary matching removes the star/start class of false positives.
  • Still capped below the others because, honestly, it's a good feedback-quality tool wearing a bias-strategy interface — full marks would mean actually tying it back to bias (the cross-check idea above), which is a larger change than a fix pass.

Cross-cutting fixes

strategies/_shared.py (new)

Four small changes that benefit all five strategies at once and cost less than fixing them independently per-file.

1. Score polarity convention

Add polarity: str = "bias_risk" to BiasStrategyOutput. Four strategies default to it; ActionabilityClassifier overrides to "quality". Stops the first aggregation pass from averaging incompatible directions.

2. Extract shared helpers

_contains_any, _sentences, and _clamp are already byte-identical across career_gap.py and hidden_ceiling.py. Move them to strategies/_shared.py alongside the new STRONG_SIGNAL_PATTERNS so the strategies can't quietly diverge.

3. Consistent matching

Standardize on word-boundary regex for single-word terms everywhere; reserve bare substring matching for genuine multi-word phrases only.

4. Test coverage parity

omission_penalty.py and lexicon_checker.py have no test files today, unlike the other three. Add test_omission_penalty.py and test_lexicon_checker.py covering: a paraphrased achievement (should not omit), a negated flagged word (should not fire), and a multi-severity feedback sample (score must not drop below the single highest severity).

Suggested rollout order

  1. Add polarity to BiasStrategyOutput and create strategies/_shared.py first — every other change depends on importing from it.
  2. LexiconChecker and ActionabilityClassifier next — smallest diffs, no cross-strategy dependency, immediate correctness win (the mean-aggregation bug is actively wrong today).
  3. HiddenCeilingEngine — swap the qualification gate to use the shared STRONG_SIGNAL_PATTERNS.
  4. OmissionPenalty — the biggest rewrite; also defines STRONG_SIGNAL_PATTERNS if not already added in step 1.
  5. CareerGapPenalty last — depends on nothing above, but its test file needs a companion update, so give it its own review pass.
